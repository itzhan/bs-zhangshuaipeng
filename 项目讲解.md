# 流浪宠物救助系统 — 项目讲解文档

> 本文档帮助你快速理解整个毕设项目的结构、技术和核心逻辑，方便应对答辩。

---

## 一、项目是做什么的？

一个**流浪动物救助管理平台**。用户可以在网站上报告流浪动物、申请领养、发布寻宠/招领信息；管理员在后台管理宠物、审核领养、管理物资和救助站。

**三个独立运行的部分：**

| 部分 | 作用 | 技术栈 | 访问地址 |
|------|------|--------|----------|
| `backend/` | 后端 API 服务 | Spring Boot 3 + MyBatis-Plus + JWT + MySQL | `http://localhost:8080/api` |
| `admin/` | 管理后台（管理员用） | Vue 3 + Naive UI（SoybeanAdmin 框架） | `http://localhost:9527` |
| `frontend/` | 用户端网站（普通用户用） | Vue 3 + Naive UI + Pinia | `http://localhost:3000` |

## 三、数据库设计（14 张表）

数据库名：`pet_rescue`，SQL 文件在 `sql/` 目录下。

### 核心表

| 表名 | 作用 | 关键字段 |
|------|------|----------|
| `sys_user` | 用户表 | username, password, role(ADMIN/STAFF/USER), status |
| `pet` | 宠物表 | name, species(种类), breed(品种), health_status, status(0待领养/1已领养等), station_id |
| `rescue_station` | 救助站 | name, address, capacity(容量) |

### 业务表

| 表名 | 作用 |
|------|------|
| `rescue_record` | 救助记录（用户报告流浪动物 → 员工处理） |
| `adoption_application` | 领养申请（用户申请 → 管理员审核） |
| `supply` | 物资管理（猫粮、药品的库存和预警） |
| `volunteer_schedule` | 志愿者排班 |
| `lost_found` | 寻宠招领信息 |
| `activity` | 爱心活动（领养日、科普讲座等） |
| `activity_registration` | 活动报名记录 |

### 辅助表

| 表名 | 作用 |
|------|------|
| `comment` | 评论 |
| `favorite` | 收藏 |
| `notification` | 系统通知（领养审核结果等推送给用户） |
| `sys_dict` | 数据字典（存储下拉菜单选项，如宠物颜色、种类等） |

### 为什么 14 个表但只有 6 个 Controller？

小的模块（评论、收藏、通知、字典、活动、救助记录、寻宠招领、排班等）都合并到了 `CommonController` 里，避免创建太多类。

## 五、核心技术点（答辩重点）

### 5.1 统一返回格式 `Result<T>`

所有接口都返回统一结构：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

对应代码 `Result.java`，包括 `success()` 和各种 `error()` 方法。

### 5.2 JWT 认证流程

```
1. 用户登录 → POST /api/auth/login（用户名+密码）
2. 后端验证通过 → 返回 JWT Token
3. 前端把 Token 存在 localStorage
4. 之后每次请求都在 Header 带上：Authorization: Bearer <token>
5. 后端 SecurityConfig 拦截请求，解析 Token 获取用户身份
```

### 5.3 权限控制

使用 Spring Security 的 `@PreAuthorize` 注解：

```java
@PreAuthorize("hasAnyRole('ADMIN','STAFF')")  // 只有管理员和员工能访问
public Result<?> create(@RequestBody Pet pet) { ... }
```

### 5.4 MyBatis-Plus 简化数据库操作

- 实体类加 `@TableName("pet")` 注解对应数据库表
- Mapper 继承 `BaseMapper<Pet>` 即可自动获得增删改查方法
- 无需手写 SQL

### 5.5 逻辑删除

实体类中的 `@TableLogic private Integer deleted;` 表示删除不是真删，而是把 `deleted` 字段设为 1，查询时自动过滤。

## 六、核心业务流程

### 6.1 救助流程

```
用户发现流浪动物 → 在用户端提交救助报告（status=0 待处理）
       ↓
员工在后台接单 → status=1 处理中
       ↓
员工把动物带回救助站 → status=2 已完成，宠物信息录入系统
       ↓
（如果虚假报告 → status=3 已取消）
```

### 6.2 领养流程

```
用户浏览宠物列表 → 点击宠物详情 → 提交领养申请
       ↓
管理员/员工在后台审核
       ↓
 ┌─ 通过（status=1）→ 宠物状态变"已领养"，通知用户
 └─ 拒绝（status=2）→ 通知用户原因
```

### 6.3 物资预警

当物资数量 ≤ 预警阈值时，系统自动提醒站长补充物资。

## 八、项目启动步骤

### 环境准备

- JDK 17+
- Maven 3.8+
- Node.js 18+ & npm/pnpm
- MySQL 8.0

### 步骤

```bash
# 1. 创建数据库并导入数据
mysql -u root -p -e "CREATE DATABASE pet_rescue DEFAULT CHARSET utf8mb4;"
mysql -u root -p pet_rescue < sql/init.sql
mysql -u root -p pet_rescue < sql/data.sql

# 2. 修改后端数据库配置
#    编辑 backend/src/main/resources/application.yml
#    改成你自己的 MySQL 用户名和密码

# 3. 一键启动（后端 + 管理端 + 用户端）
bash start.sh

# 4. 停止所有服务
bash stop.sh
```

## 十、答辩常见问题准备

| 问题 | 回答要点 |
|------|----------|
| 项目用了什么技术？ | 前后端分离，后端 Spring Boot 3 + MyBatis-Plus + JWT + MySQL，前端 Vue 3 + Naive UI，管理端基于 SoybeanAdmin 框架 |
| 怎么实现用户认证的？ | JWT 无状态认证，登录返回 Token，前端每次请求带 Token，后端 Spring Security 过滤器验证 |
| 怎么做权限控制的？ | 数据库 `role` 字段区分角色，后端用 `@PreAuthorize` 注解控制接口权限 |
| 为什么用 MyBatis-Plus？ | 简化 CRUD 操作，内置分页插件、逻辑删除、自动填充时间等功能 |
| 前后端怎么交互的？ | 前端通过 Axios 发送 HTTP 请求到后端 RESTful API，统一返回 `{code, message, data}` 格式 |
| 领养流程怎么实现的？ | 用户提交申请 → 后台审核 → 通过后自动更新宠物状态为已领养并发送通知 |
| 怎么部署的？ | 提供 `start.sh` 一键启动脚本，也支持 Docker Compose 部署 |
